# Programar para enseñar — Python e IA generativa para Humanidades Digitales

Materiales del curso de **formación docente** para la Unidad de Formación **EH1023 — Programación para Humanidades Digitales** del Tecnológico de Monterrey.

Diseñado e impartido por la **Pontificia Universidad Javeriana — Facultad de Ingeniería**.

> El objetivo del curso no es convertir a los participantes en programadores expertos, sino en **facilitadores competentes**: docentes capaces de comprender la lógica computacional, leer y explicar código básico en Python, y usar la inteligencia artificial generativa de forma **crítica y pedagógica** en sus aulas.

---

## A quién está dirigido

Profesores de humanidades, educación, comunicación, artes y ciencias sociales, con **niveles muy diversos** de experiencia. El curso **se trabaja desde cero**: nadie necesita haber programado antes. Quienes ya tienen experiencia encontrarán valor en el **enfoque metodológico** —cómo enseñar, calibrar y evaluar— más que en la sintaxis.

---

## Cómo usar estos materiales

### Opción principal — Google Colab (sin instalar nada)

1. Entra a [Google Colab](https://colab.research.google.com/).
2. Abre el notebook (`.ipynb`) de la carpeta del día, ya sea subiéndolo (**Archivo → Subir cuaderno**) o desde GitHub (**Archivo → Abrir cuaderno → GitHub**).
3. Ejecuta cada celda con **Shift + Enter**.

Todo el curso usa cuadernos **`.ipynb`** y funciona en cualquier sistema operativo (incluido Mac) directamente desde el navegador.

### Opción alterna (plan B) — Python local con VS Code

Para quienes prefieran un entorno local —o trabajar sin conexión—, el **primer día** instalaremos Python y Visual Studio Code con apoyo del monitor. La guía paso a paso está en [`dia-1/`](dia-1/).

### Herramientas de IA generativa

El curso utiliza únicamente las herramientas **autorizadas por el Tec de Monterrey**:

- **TecGPT** (herramienta institucional propia)
- **Gemini**
- **Copilot**

No se emplean ChatGPT, Claude, Chartmetric ni ManyChat.

## Una idea que atraviesa todo el curso

Como docentes, ustedes serán quienes **delimiten los talleres de sus estudiantes**. Una parte central de enseñar programación es **calibrar la dificultad**: proponer retos alcanzables y graduados, y evitar ejercicios extremadamente difíciles que, al fallar, solo consigan **desanimar** a quien está empezando. A lo largo del curso volveremos sobre cómo dosificar el reto para que cada estudiante avance con confianza.
